Terraform : IaC : Infrastructure as a code
HCL : hashicorp configuration language

development => testing => deployment servers

Key features

1. Declarative syntax : define what infrastructure you want, not how to create it
2. Idempotent : running the same code repeatedly, produces the same results
3. Cloud agnostic : supports major cloud providers like AWS, Azure & GCP
4. Modular : reusable code through modules
5. Execution plan : Terraform shows what it will do before applying changes

Core workflow

1. terraform init : initialize the working directory
2. terraform plan : shows what changes will be made
3. terraform apply : applies the changes to your infrastructure
4. terraform destroy : destroys the infrastructure

Key Components

- Providers : Connect terraform to cloud platforms (AWS, Azure, GCP)
- Resources : The actual components (VM's, database, storage) you want to manage
  resource <PROVIDER> <NAME>{}
- Variables : parametrize the configurations
- State files : Track the current state of your infrastructure
- Modules : reusability : packages of terraform configuration files
